# Pivotly V2 Frontend Changes

## Overview
The V2 Frontend upgrade replaces the generic top-down reading flow with an interactive, tabbed dashboard. The focus shifts from "reading a report" to "exploring intelligence."

## 1. Type Definitions (`frontend/src/types/report.ts`)
The TypeScript definitions must be updated to support the `VentureReportV2` structure. 
A `schema_version` discriminator should be added to the base `Report` type.

```typescript
export interface Report {
  id: string;
  status: 'PENDING' | 'SCRAPING' | 'GENERATING' | 'COMPLETED' | 'FAILED';
  schema_version: number; 
  // ...
}
```

## 2. Component Architecture

### The `ReportView` Container
Instead of rendering a single vertical scroll of sections, the `ReportView` will use a Tabbed interface.
- **Tabs:** Overview, Market, Competitors, Moat, Risks, Scorecard.

### New Components

1. **`EvidenceCard.tsx`**
   - Reusable component used across all tabs.
   - Takes `claim`, `quote`, `source_url`, and `confidence`.
   - Renders the claim in bold, the quote in italics, and a clickable external link to the source.
   - Shows a color-coded confidence badge (Green = High, Yellow = Medium).

2. **`ScorecardView.tsx`**
   - Renders the deterministic scores generated by the Python backend.
   - Displays a large radial progress bar for the `overall_score`.
   - Displays linear progress bars for `market_score`, `competition_score`, `moat_score`, and `execution_risk_score`.
   - Includes a tooltip or collapsible section that reads the `scoring_rationale` dictionary to explain *why* the backend assigned that exact score.

3. **`MoatMeter.tsx`**
   - A visual gauge showing defensibility.
   - Renders boolean checks (e.g., `Network Effects: ❌`, `Data Advantage: ✅`).

4. **`CompetitorGrid.tsx`**
   - Displays the array of competitors as a masonry grid of cards.
   - Clearly highlights the `copy_risk` string and `threat_level` badge (Red/Orange/Yellow).

5. **`ContrarianPanel.tsx` (Risks Tab)**
   - Styled with a darker, more serious aesthetic (e.g., slight red tint).
   - Lists the "Why This Could Fail", "Top Failure Reasons", and "Critical Assumptions".

## 3. Backward Compatibility Support
The frontend must not break when a user clicks on an older report generated under V1.

**Implementation Strategy:**
In `ReportPage.tsx`:
```tsx
if (report.schema_version === 1 || !report.schema_version) {
  return <LegacyReportView report={report.report_json as VentureReportV1} />;
} else {
  return <V2ReportDashboard report={report.report_json as VentureReportV2} />;
}
```

## 4. Design Aesthetics
- Ensure the UI feels like a premium Bloomberg Terminal or professional VC dashboard, rather than a generic text document.
- Use glassmorphism for Evidence Cards.
- Ensure all quantitative scores are animated upon load.
