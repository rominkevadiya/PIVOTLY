# Pivotly V2 Schema Design

## Overview
This document outlines the flattened, modular JSON schema design for Pivotly's V2 Venture Intelligence reports (`VentureReportV2`). The goal is to minimize nesting, reduce Pydantic validation failures, enforce evidence citations, and strip hallucinated quantitative scores from LLM generation.

## 1. Reusable Primitives

### `Evidence` Schema
Every major qualitative claim generated by an agent MUST be supported by an `Evidence` object.
```python
class Evidence(BaseModel):
    claim: str = Field(description="The factual claim being made")
    source_url: str | None = Field(description="URL to the source backing this claim, if available")
    quote: str | None = Field(description="Direct quote or specific data point from the source")
    confidence: str = Field(description="High, Medium, or Low based on source quality")
```

## 2. Agent Output Schemas

### `ResearchContext`
Generated by the Research Service (No opinions, only facts).
```python
class ResearchContext(BaseModel):
    market_size_estimate: str
    growth_rate_estimate: str
    target_audience: list[str]
    identified_competitors: list[str]
    regulatory_concerns: list[str]
    evidence_list: list[Evidence]
```

### `CompetitorAnalysis`
Generated by the Competitor Service.
```python
class Competitor(BaseModel):
    name: str
    website: str | None
    threat_level: str = Field(description="High, Medium, Low")
    copy_risk: str = Field(description="Explanation of how easily they could copy the user's idea")
    strengths: list[str]
    weaknesses: list[str]

class CompetitorAnalysis(BaseModel):
    competitors: list[Competitor]
    market_concentration: str = Field(description="Fragmented, Consolidated, Monopoly")
    evidence_list: list[Evidence]
```

### `MoatAnalysis`
Generated by the Moat Service.
```python
class MoatAnalysis(BaseModel):
    moat_type: str = Field(description="Brand, Network Effects, Switching Costs, Cost Advantage, None")
    defensibility_explanation: str
    copy_difficulty: str = Field(description="High, Medium, Low")
    network_effects_present: bool
    data_advantage_present: bool
    evidence_list: list[Evidence]
```

### `ContrarianAnalysis`
Generated by the Contrarian Service (Sequoia Partner persona).
```python
class ContrarianAnalysis(BaseModel):
    top_failure_reasons: list[str]
    critical_assumptions: list[str]
    largest_unknowns: list[str]
    execution_risks: list[str]
```

## 3. Deterministic Scorecard Schema
Generated entirely by the Python Backend (`scoring_service.py`), NOT the LLM.
```python
class Scorecard(BaseModel):
    market_score: int = Field(ge=0, le=10)
    competition_score: int = Field(ge=0, le=10)
    feasibility_score: int = Field(ge=0, le=10)
    monetization_score: int = Field(ge=0, le=10)
    moat_score: int = Field(ge=0, le=10)
    execution_risk_score: int = Field(ge=0, le=10)
    overall_score: int = Field(ge=0, le=100)
    scoring_rationale: dict[str, str] = Field(description="Explanation of how scores were calculated")
```

## 4. Final V2 Report Schema
The master schema stored in the database `report_json` and sent to the frontend.

```python
class VentureReportV2(BaseModel):
    # 1. Executive Summary
    executive_summary: str
    
    # 2. Market Intelligence
    market_intelligence: ResearchContext
    
    # 3. Competitor Intelligence
    competitor_intelligence: CompetitorAnalysis
    
    # 4. Moat Analysis
    moat_analysis: MoatAnalysis
    
    # 5. Opportunity Gaps
    opportunity_gaps: list[str]
    
    # 6. Failure Analysis
    failure_analysis: ContrarianAnalysis
    
    # 7. Strategic Synthesis (Replacing SWOT)
    why_this_could_win: list[str]
    why_this_could_fail: list[str]
    what_must_be_true: list[str]
    
    # 8. Deterministic Metrics
    scorecard: Scorecard
    
    # 9. Recommendation
    recommendation: str = Field(description="Build, Pivot, Research Further, Avoid")
    recommendation_rationale: str
```

## Design Rationale
- **Flattened Structure:** Removed deeply nested SWOT and Investor Verdict objects. 
- **Modularity:** Each agent outputs a specific schema (`ResearchContext`, `CompetitorAnalysis`), which is directly injected into the master `VentureReportV2` by the orchestration layer. This prevents a single LLM validation failure from crashing the entire report.
- **Evidence Driven:** The inclusion of `evidence_list` at the agent level forces the LLM to cite URLs and quotes for its claims before that section is approved.
